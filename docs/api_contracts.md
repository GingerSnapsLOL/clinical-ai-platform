## API Contracts

This document summarizes the main API contracts defined in `services/shared/schemas_v1.py`.  
The source of truth is always the Pydantic models in that module.

---

## Conventions

- All request/response models include a `trace_id` field for end-to-end traceability.
- All services use a shared `Status` literal:
  - `"ok"` – successful operation.
  - `"error"` – error envelope with `ErrorInfo`.
- Errors are represented as:
  - `ErrorInfo` with:
    - `code: str`
    - `message: str`
    - `details: dict | None`

---

## Health

### Response

- **Model**: `HealthResponse`
- **Fields**:
  - `status: "ok"`
  - `service: str` – service name.
  - `version: str` – currently `"0.1.0"`.

All services expose `GET /health` returning `HealthResponse`.

---

## Gateway / Orchestrator – `/v1/ask`

### Request (internal)

- **Model**: `AskRequest`
- **Fields**:
  - `trace_id: str` – UUID generated by gateway when omitted by the client.
  - `mode: Mode` – `"strict"` or `"hybrid"` (currently only strict behavior is implemented).
  - `note_text: str` – clinical note text.
  - `question: str` – user question.
  - `user_context: dict | None` – optional extra context.

### Response

- **Model**: `AskResponse`
- **Fields**:
  - `status: Status` – `"ok"` or `"error"`.
  - `trace_id: str`
  - `pii_redacted: bool` – whether PII redaction was applied.
  - `answer: str` – synthesized answer (LLM or deterministic fallback).
  - `sources: list[SourceItem]` – evidence passages.
  - `entities: list[EntityItem]` – extracted entities from NER.
  - `risk: RiskBlock | None` – risk score + explanation (currently stubbed).
  - `warnings: list[str]` – non-fatal pipeline warnings.
  - `error: ErrorInfo | None` – populated when `status == "error"`.

### Supporting types

- `SourceItem`
  - `source_id: str`
  - `title: str | None`
  - `url: HttpUrl | None`
  - `snippet: str | None`
  - `score: float`
  - `metadata: dict | None`

- `CitationItem`
  - `source_id: str`
  - `title: str | None`
  - `url: HttpUrl | None`

- `EntityItem`
  - `type: str`
  - `text: str`
  - `start: int`
  - `end: int`
  - `confidence: float | None`

- `FeatureContribution`
  - `feature: str`
  - `contribution: float`

- `RiskBlock`
  - `score: float`
  - `label: "low" | "medium" | "high"`
  - `explanation: list[FeatureContribution]`

---

## PII Service – `/v1/redact`

### Request

- **Model**: `RedactRequest`
- **Fields**:
  - `trace_id: str`
  - `text: str`

### Response

- **Model**: `RedactResponse`
- **Fields**:
  - `status: Status`
  - `trace_id: str`
  - `redacted_text: str`
  - `spans: list[PIISpan]`
  - `error: ErrorInfo | None`

- `PIISpan`
  - `type: str` – entity type (e.g. PERSON, PHONE_NUMBER, EMAIL_ADDRESS).
  - `start: int`
  - `end: int`
  - `replacement: str`
  - `confidence: float | None`

---

## NER Service – `/v1/extract`

### Request

- **Model**: `ExtractRequest`
- **Fields**:
  - `trace_id: str`
  - `text: str`

### Response

- **Model**: `ExtractResponse`
- **Fields**:
  - `status: Status`
  - `trace_id: str`
  - `entities: list[EntityItem]`
  - `error: ErrorInfo | None`

---

## Retrieval Service – `/v1/retrieve`

### Request

- **Model**: `RetrieveRequest`
- **Fields**:
  - `trace_id: str`
  - `query: str`
  - `top_k: int` – number of points to retrieve from Qdrant (default 50).
  - `top_n: int` – maximum number of passages to return when rerank is disabled (default 8).
  - `rerank: bool` – whether to apply cross-encoder reranking (default `True`).

### Response

- **Model**: `RetrieveResponse`
- **Fields**:
  - `status: Status`
  - `trace_id: str`
  - `passages: list[PassageItem]`
  - `error: ErrorInfo | None`

- `PassageItem`
  - `source_id: str`
  - `text: str`
  - `score: float`
  - `metadata: dict | None`

---

## Retrieval Service – `/v1/ingest`

The ingest API is defined in `services/retrieval-service/app/main.py` using local Pydantic models (not re-exported from `schemas_v1.py`), but its behavior is:

- **Request**:
  - `documents: list[{ doc_id: str, text: str, metadata: dict }]`
- **Response**:
  - `chunks_inserted: int`

---

## Scoring Service – `/v1/score`

### Request

- **Model**: `ScoreRequest`
- **Fields**:
  - `trace_id: str`
  - `entities: list[EntityItem]` – entities from NER.
  - `structured_features: dict[str, Any]` – additional tabular/structured features (currently unused).

### Response

- **Model**: `ScoreResponse`
- **Fields**:
  - `status: Status`
  - `trace_id: str`
  - `score: float`
  - `label: "low" | "medium" | "high"`
  - `explanation: list[FeatureContribution]`
  - `error: ErrorInfo | None`

**Implementation note**: as of now, `scoring-service` returns a fixed score/label and sample feature contributions (stub implementation).

---

## LLM Service – `/v1/generate`

The LLM service defines its own request/response models in `services/llm-service/app/main.py` (not via `schemas_v1.py`):

- **Request** (GenerateRequest)
  - `trace_id: str`
  - `prompt: str`
  - `max_tokens: int` – max new tokens (default 512).
  - `temperature: float` – sampling temperature (default 0.2).

- **Response** (GenerateResponse)
  - `status: Status` – `"ok"` on success.
  - `trace_id: str`
  - `text: str` – generated completion.
  - `usage: dict[str, int]` – `prompt_tokens`, `completion_tokens`, `total_tokens`.

The orchestrator uses `services.shared.llm_client.LLMClient` to call this API.

---

## Error handling patterns

- Services should:
  - Use `status="ok"` and omit `error` on success.
  - Use `status="error"` and populate `error: ErrorInfo` when returning a logical error envelope.
- The gateway wraps orchestrator/network errors in a JSON error body containing:
  - `status: "error"`
  - `trace_id: str`
  - `error: ErrorInfo`

For the authoritative details and any future changes, refer to `services/shared/schemas_v1.py` and individual service `app/main.py` files.
